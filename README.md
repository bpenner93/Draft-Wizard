# Draft Wizard 🏈

Fantasy hub: a live draft assistant + the in-season weekly pages (season
rankings, weekly DFS board, lineup builder, expert compare, Vegas watch) —
one Streamlit app, phone-friendly.

## Pages
- **draft app** (main) — the live draft assistant (below)
- **📊 Season Rankings** — our season-long board vs Clay/ECR vs ADP (`data/draft_board.json`)
- **🎯 Weekly DFS** — the week's DFS board (proj/salary/value/own/floor-ceil/Vegas),
  ours-vs-market divergence (the sanity-flag list), games board (`data/weekly_bundle.json`)
- **🧱 Lineup Builder** — DK classic lineups in-app: GPP mirrors the validated profile
  build (high-total stacks, ~55% bring-back, de-chalk hinge), Cash = max floor-proj;
  locks/excludes/exposure; DK CSV download; manual build + auto-complete
- **🔍 Expert Compare** — paste/upload Boone/Clay/anyone's weekly list, diff vs our
  rank + the market; flags players they rank that we zero (availability catches)
- **📈 Vegas Watch** — bundled baseline lines vs LIVE ESPN lines on demand; implied-total
  movement alerts + affected players (big move = re-run the pipeline before lock)

## The draft assistant

A live fantasy-draft assistant (redraft + rookie). Mark players off as they're
drafted and it recomputes, after every pick:

- **Best pick** — value (VOR) over replacement in *your* league's scoring/roster
- **Tiers left** — where the value cliffs are per position
- **Survival odds** — Monte-Carlo P(a player is still there at your next pick)
- **Draft-now-vs-wait** — the player whose value you'd lose by waiting × your roster need
- **Opponent reads** — each team's roster shape and likely position runs before your pick

Works for **Sleeper** (auto-sync picks), **ESPN**, and **in-person** drafts
(manual mark-off). Built on a consensus board = our projection model + Waldman
blend, Clay ECR, and live Fantasy Football Calculator ADP.

## Run locally
```bash
pip install -r requirements.txt
streamlit run draft_app.py
```

## Use on your phone (Streamlit Community Cloud — free)
1. Put **this `draft_wizard/` folder** in its own GitHub repo (it's self-contained:
   `draft_app.py`, `draft_engine.py`, `draft_names.py`, `data/draft_board.json`,
   `requirements.txt`). Do **not** add the rest of the projection pipeline.
2. Go to <https://share.streamlit.io> → **New app** → pick the repo →
   main file `draft_app.py` → **Deploy**.
3. Open the resulting `https://…streamlit.app` URL on your phone. Bookmark it.
   During a draft, tap players off (or paste your Sleeper Draft ID to auto-sync).

## Sleeper live-sync
In the sidebar, paste your **Draft ID** — the number in your draft URL
`sleeper.com/draft/nfl/<DRAFT_ID>` — and tap **Pull picks from Sleeper**. It
replaces the board state with the real draft so far. Re-tap to refresh.

## Refreshing the board (do this before draft season / weekly)
The board data (`data/draft_board.json`) is generated by the **private** pipeline:
```bash
python adp_ingest.py          # live ADP + stdev (FFC)
python walter_ingest.py       # scrape WalterFootball season ranks (QB/RB/WR/TE + proj pts)
python consensus.py           # blend ours + Waldman + Clay/Boone/Walter ECR + ADP
python export_draft_board.py  # -> draft_wizard/data/draft_board.json
```
Commit the refreshed `draft_board.json` to the app repo to update the phone app.

## Weekly refresh (in-season, Tue/Wed + after big news)
The weekly pages read `data/weekly_bundle.json`, generated by the private pipeline:
```bash
# after the weekly pipeline run (weekly_engine/season_rankings + DK salary import):
python export_weekly_bundle.py --season 2026 --week N --fetch-odds
```
Commit the refreshed `weekly_bundle.json` (plus any saved `data/experts/*.json`)
to the app repo to update the phone app. Locally the pages pick it up on refresh.

Weekly loop: import DK salaries → run pipeline → export bundle → ingest experts
on the 🔍 page and chase the big gaps to a REASON (fix facts, not numbers) →
watch 📈 Vegas for moves → rebuild lineups. Contest selection guidance (big
casual GPPs, soft double-ups) lives in the pipeline: `soft_contest_finder.py`.

## Tuning the pick logic
The recommendation knobs live at the top of `draft_engine.py`:
`W_SCARCITY`, `W_NEED`, `KDST_LATE_ROUNDS`, the tier-gap threshold, and
`need_scores`. These are what we iterate to "tune in" the pick analyzer.
